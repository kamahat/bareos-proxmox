## The GID problem (and the fix)

`bareos-fd` runs with `Gid=124` (the `bareos` group). `pmxcfs` вЂ” the Proxmox cluster file
system daemon вЂ” validates every IPC connection via `SO_PEERCRED` and **requires `gid=0`**.
Without a fix, `vzdump` launched by the plugin inherits `Gid=124` and fails immediately:

```
ipcc_send_rec[1] failed: Unknown error -1   (x3 then exit 255, then 5-minute LogPipe timeout)
```

The wrapper `vzdump-bareos` (`/usr/bin/vzdump-bareos`) fixes this by calling
`os.setregid(0, 0)` + `os.setgroups([0])` before `os.execv("/usr/bin/vzdump", ...)`.
This invariant is **verified by an automated test on every build** (`tests/test_wrapper.py`).

## Patch summary

`patch/bareos_fd_proxmox.patch` adds to the upstream source:

| Change | Reason |
|--------|--------|
| `mode` option (default `stop`) | Passed as `--mode` to vzdump; supports `stop`, `snapshot`, `suspend` |
| Calls `vzdump-bareos` instead of `vzdump` | Wrapper handles the mandatory GID fix |
| `start_new_session=True` in `Popen` | Isolates vzdump's process session from bareos-fd |
